🚀 AutoMaster 全能自动化助手 (V2.0)
AutoMaster 是一个基于 Python 开发的轻量级、高智能的桌面自动化工具。它集成了动作录制、智能图像识别转换、拟人化操作回放以及强大的容错修复机制。

与传统的按键精灵不同，AutoMaster 模拟人类的操作轨迹（贝塞尔曲线移动、随机抖动、非线性速度），并具备“找不到图时请求人工介入并自动修复”的能力。

✨ 核心功能
🕹️ 动作录制：精准记录鼠标移动、点击（单击/双击/中键）、拖拽、滚轮及键盘操作。

👁️ 智能转换 (ModifyEye)：将坐标导向的脚本转换为图像识别导向。采用可视化引导，用户只需“画圈”即可截取目标按钮，无需写代码。

🤖 拟人化回放：

随机性：操作坐标带有微小抖动，时间间隔带有随机波动，防止被检测为机器人。

平滑移动：使用缓动算法模拟人手移动轨迹。

拖拽优化：完美还原鼠标拖拽操作。
双击保护：智能识别双击逻辑，确保系统正确响应。

🛡️ 强大的容错机制：

智能避让：找不到图时，自动移开鼠标重试。

人工接管：重试失败后暂停并提示用户，用户画圈指出位置后，程序自动继续。

自动修复：支持一键更新失效的截图文件，无需重新录制。

🎮 便捷 GUI：提供图形界面管理脚本，支持 F12 隐藏/呼出。

📂 项目结构
AutoMaster/
├── main.py           # [入口] 程序主入口，负责GUI和调度
├── config.py         # [配置] 全局参数（速度、阈值、路径等）

├── recorder.py       # [模块] 录制器
├── modify_eye.py     # [模块] 智能转换器
├── imitate.py        # [模块] 播放器
├── scripts/          # [数据] 存放录制的脚本文件 (.txt)
└── images/           # [数据] 存放截图资源 (.png)

❓ 常见问题及解决方法 (Troubleshooting)
Q1: 程序启动报错 ModuleNotFoundError
原因: 缺少依赖库。

解决: 确保已执行 pip install pyautogui pynput opencv-python pillow。

Q2: 按快捷键（如 Alt+F9）没有反应
原因:

输入法冲突或被其他软件占用。

程序没有以管理员权限运行（某些游戏或系统界面会屏蔽普通权限的模拟输入）。

解决:

尝试以管理员身份运行 CMD 或 IDE。

检查是否有其他软件占用了该热键。

Q3: 智能转换(F8)时，截图报错“范围太小”
原因: 画框时鼠标移动距离极短，或者画成了一条直线。

解决:

尽量画一个对角线（矩形）。

新版本已加入自动膨胀机制，只要有一点移动意图，程序会自动修正为 10x10 的矩形，通常只需稍微画大一点即可。

Q4: 智能播放(F2)时，找不到图片怎么办？
AutoMaster 具备自修复功能：

程序会先尝试移开鼠标并重试。

若失败，屏幕左上角会提示红色警告。

此时人工介入：移动鼠标圈出当前屏幕上该按钮的位置，停住不动。

程序识别到你的指引后，会再次搜索该区域。

若仍失败，提示按 [Space] 空格键。

程序会自动截图你刚才圈出的区域，覆盖旧图片，并继续执行。下次运行就不会报错了。

Q5: 拖拽操作回放不准确
原因: 早期版本的拟人化算法干扰了拖拽轨迹。

解决: 确保使用的是 V2.0 代码。新版加入了拖拽锁定机制，拖拽时会自动切换为线性移动模式，能够精准还原拖拽距离。

Q6: 录制时滚轮无法滚动或回放无效
原因: 不同系统对滚轮数值的定义不同。

解决: 代码中已对 scroll 进行了倍率放大 (*100)。如果在你的系统上滚动太快或太慢，请在 imitate.py 中搜索 pyautogui.scroll 并调整乘数。

Q7: 为什么双击有时候变成了两次单击？
原因: 系统双击判定时间极短（通常 <500ms），如果为了拟人化加了太多延迟，会导致判定失败。

解决: 配置文件 config.py 中有 DOUBLE_CLICK_THRESHOLD。代码会自动检测短间隔操作，并在双击时强制压缩时间间隔（0.05s-0.12s），确保双击成功。

⚠️ 注意事项
屏幕缩放: 如果你的 Windows 屏幕缩放比例不是 100%（例如 125% 或 150%），pyautogui 的截图和坐标可能会偏移。建议在 100% 缩放下使用，或配置 Python 程序的 DPI 感知。

遮挡问题: 图像识别依赖屏幕可见内容。播放时请确保目标窗口未被其他窗口完全遮挡（虽然程序有鼠标避让功能，但无法穿透窗口）。

紧急停止: 遇到任何失控情况，请狂按 F4 键！
